# Веб-интерфейс

Технологии: Vue 3, TypeScript и Vite. Это браузерный интерфейс, а не нативное приложение.

Разделы:

1. «Сейчас» — состояние устройства и синхронизации.
2. «Карта» — исходный и отфильтрованный трек, а также ведение по маршруту.
3. «Треки» — файлы, создание нового трека, продолжение загрузки и экспорт.
4. «Фильтры» — профили фильтрации, выполняемые только в браузере.
5. «Настройки» — время бодрствования и Deep Sleep, экран и BLE при таймерном пробуждении, соблюдение сна при подключённом BLE.

Записанные блоки сохраняются в IndexedDB без изменения. Представление после
фильтрации является производным и может быть пересчитано с другим профилем.

Начальный профиль фильтра содержит три независимо отключаемых порога: минимум
спутников, максимум HDOP и максимальный скачок между соседними принятыми точками.
Нулевое значение отключает порог; все значения по умолчанию равны нулю, поэтому
после первой синхронизации отображаются все исходные точки.

Экспорт GPX и GeoJSON всегда строится из текущего производного представления с
применённым профилем фильтра. Исходный `.c6t` и блоки IndexedDB при этом не
изменяются.

Карта принимает GPX (`trkpt`, `rtept`, `wpt`) и GeoJSON (`LineString`, `Point`,
`MultiPoint`, `FeatureCollection`). Для маршрута только из точек порядок следования
совпадает с порядком точек в импортированном файле.

## Мобильная вёрстка

Основной целевой телефон — Oppo Find X7 Ultra. Страница содержит обычный mobile
viewport с `viewport-fit=cover`, учитывает безопасные зоны экрана через
`env(safe-area-inset-*)` и не использует внешний шрифт. На ширине до 600 CSS-пикселей:

- вкладки размещаются сеткой по три кнопки;
- карточки телеметрии размещаются в две колонки;
- формы и списки переходят в одноколоночный режим;
- минимальная высота интерактивного элемента составляет 44 пикселя;
- элементы управления картой занимают доступную ширину без горизонтальной прокрутки.

Контрольные viewport для браузерной проверки: `412×915` и `360×800` CSS-пикселей.

## Подключение Web Bluetooth

Подключение повторяет проверенную последовательность предыдущей версии, но работает
с UUID нового двоичного протокола:

1. Выбор устройства выполняется с `acceptAllDevices`, а UUID сервиса передаётся в
   `optionalServices`. Это позволяет Android показать трекер даже при неполной
   рекламе пользовательского UUID.
2. Ранее разрешённый `C6 Tracker v2` восстанавливается через
   `navigator.bluetooth.getDevices()` без повторного системного диалога.
3. После GATT-подключения всегда заново выполняются поиск сервиса и характеристик,
   включая случай, когда Android уже считает GATT соединённым.
4. Записи управляющей характеристики сериализуются и по возможности выполняются
   через `writeValueWithResponse`.
5. После Deep Sleep выполняется повторная попытка подключения каждые три секунды.

Web Bluetooth требует Chrome/Chromium и безопасный контекст. Опубликованная
GitHub Pages-версия работает по HTTPS.
