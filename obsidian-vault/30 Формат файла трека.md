# Формат файла трека

Расширение: `.c6t`. Порядок байтов: little-endian.

Текущая реализация использует 40-байтовый заголовок `C6T2`, после которого идут
фиксированные записи по 48 байт. Каждая запись защищена собственным CRC32. После
оборванной записи восстановление отбрасывает только повреждённый хвост файла.
Сжатие точек при передаче относится к транспортному протоколу и не изменяет
исходный файл.

Заголовок содержит идентификатор трека, идентификатор устройства, время создания,
настройки бодрствования и сна, флаги и CRC32. Нормативные упакованные структуры
описаны в `firmware/include/binary_format.h`.

Один `track_id` — один файл вида `track_NNNNNN.c6t`. Перезагрузка и Deep Sleep не
создают новый файл. Новый файл создаётся только явной командой «Новый трек» либо
при первом запуске на пустой SD-карте.
