# Журнал реализации

## 2026-07-15

- Создан чистый проект `c6_tracker_v2`.
- Из старого проекта взяты только контакты платы и последовательность инициализации контроллера дисплея.
- Определены заголовок файла C6T2 и фиксированная запись точки с CRC32.
- Реализована дозапись текущего файла, восстановление повреждённого хвоста и накопление необработанного расстояния.
- Добавлен минимальный экран на Arduino_GFX без LVGL и дополнительных шрифтов.
- Создан новый двоичный BLE-сервис со статусом, настройками, созданием нового трека и уведомлениями о точках.
- После уточнения требований удалены внутренний планировщик PSMOO и настройка частоты координат.
- GPS оставлен в штатном режиме 1 Гц во время бодрствования; перед общим Deep Sleep отправляется `UBX-RXM-PMREQ`.
- Создан начальный веб-интерфейс Vue с подключением по Web Bluetooth, картой, импортом линейных и точечных маршрутов и кнопкой очистки маршрута.
- Реализованы список файлов и возобновляемая bulk-передача блоками по 180 байт с CRC32 и накопительным ACK.
- Исходные блоки сохраняются транзакционно и без изменения в IndexedDB; добавлены сборка и проверка полного файла C6T2 перед показом на карте.
- Добавлено автоматическое переподключение к уже выбранному трекеру после планового Deep Sleep.
- Добавлено независимое PackBits-сжатие bulk-блоков с raw-резервом для несжимаемых данных.
- Реализован экспорт отфильтрованного представления в GPX 1.1 и GeoJSON.
- Исправлена публикация веб-интерфейса: GitHub Pages переведён с Jekyll корня
  репозитория на production-сборку `web/dist` через GitHub Actions.
- Web Bluetooth адаптирован по проверенной схеме старой версии: свободный системный
  выбор устройства, `optionalServices`, восстановление разрешённого трекера,
  последовательные записи с подтверждением и устойчивое переподключение.
- Добавлена мобильная вёрстка для Oppo Find X7 Ultra с mobile viewport, safe-area,
  крупными touch-элементами и адаптивными картой, вкладками, формами и списками.
- Экспорт GPX и GeoJSON сделан явным для текущего и архивных треков; при неполной
  локальной копии экспорт автоматически выполняет возобновляемую BLE-синхронизацию.
- В настройки возвращены профили «Пешком», «Авто», «Эконом», пользовательские
  профили, автоподключение и телефонные пороги фильтрации.
- Реализована очередь настроек в `localStorage`: пакет можно сохранить во время
  сна устройства, после подключения он отправляется и подтверждается автоматически.
- Добавлено измерение аккумулятора, поле напряжения в BLE-статусе и совместимое
  расширение C6T2 записями аккумулятора без фиктивных GPS-точек.
- Усилено выключение подсветки перед Deep Sleep: отключение PWM, `GPIO LOW` и hold.
- «Карта» и «Треки» объединены; добавлены линия, точки, набор полей подписи и
  полноэкранный режим с CSS-резервом.
- Экспорт переведён на все исходные точки и полную телеметрию в GPX/GeoJSON.
- В начало настроек добавлены график аккумулятора и список применённых настроек.
- Вебка превращена в устанавливаемую PWA; просмотренные OSM-тайлы кэшируются без
  массовой предзагрузки.
- Добавлена Android-обёртка Capacitor с нативным BLE. Старый проект
  `D:\\Android_projects\\c6` изучен только как справочник: он реализует отдельный
  Compose-интерфейс и прежний текстовый протокол, поэтому смешивать его с v2 нельзя.

## Пакет изменений 2026-07-16

- Параметр бодрствования переопределён как ожидание первой точки; добавлены
  `pointsBeforeSleep=3` и аварийный 30-секундный тайм-аут после первой точки.
- BLE включается при каждом пробуждении и работает до самого Deep Sleep. Активный
  USB Serial/JTAG host безусловно запрещает сон.
- Кнопка GPIO5 принудительно включает экран на 30 секунд; надписи `ALT` и `BT` на
  физическом дисплее увеличены.
- Перед остановкой UART линия TX удерживается в высоком уровне, затем GPS получает
  `UBX-RXM-PMREQ backup+force`, чтобы окончание UART не разбудило модуль фронтом.
- Добавлена включаемая с экрана «Сейчас» диагностика NMEA GGA без записи в трек.
- Реализованы четыре картографические подложки, их офлайн-кэш, новые цвета трека,
  безопасное положение полноэкранных кнопок и подробные статусы локальных файлов.
- Текущий C6T2 автоматически догружается во внутреннее хранилище с последнего
  смещения; счётчик загрузки обновляется после каждого BLE-блока.
- Экспорт Android сохраняет GPX/GeoJSON в `Download/C6 Tracker` через MediaStore;
  исходные C6T2 остаются только во внутреннем хранилище приложения.
- Vitest: 5 файлов, 16 тестов — успешно. Vue/TypeScript и production Vite — успешно.
- Arduino-ESP32 3.3.10: сборка ESP32-C6 успешна, flash 886 872 байта (26% раздела),
  глобальная RAM 22 520 байт (6%). Android debug APK после синхронизации ресурсов
  собран успешно; ADB-устройство на момент проверки не подключено.
- Коммит `d7cce66` опубликован в `main`, workflow GitHub Pages завершился успешно.
  Опубликованная страница проверена на viewport `412×915`: горизонтального
  переполнения нет, полноэкранные `+`, `−` и «Свернуть» имеют верхнюю координату
  58 px, CARTO Dark Matter действительно загружает свои тайлы, консоль чистая.
